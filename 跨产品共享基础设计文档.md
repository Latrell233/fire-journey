## 二、设计系统 (Design Tokens)

### 2.1 品牌调性

**温润极简风 (Warm Minimalist)** — 参考 Linear + Headspace

- 关键词：安静、信赖、呼吸感、品质、不焦虑
- 浅色/暖色底、大圆角、柔和阴影、克制的色彩
- 金融数据需要信任感，情感连接需要温暖感

### 2.2 色板

#### 基础色

| Token | 色值 | 用途 |
|---|---|---|
| `--color-primary` | `#c8a882` (暖铜棕) | 主强调色 — 按钮、选中态、进度条、关键数据 |
| `--color-surface` | `#f9f6f0` (暖杏白) | 页面底色 |
| `--color-card` | `#ffffff` | 卡片背景 |
| `--color-muted` | `#f0ebe4` (羊毛色) | 输入框底色、分隔区域 |
| `--color-text` | `#2c2c2c` (墨灰) | 主文字色 |
| `--color-text-secondary` | `#6b6258` | 次要文字 |
| `--color-text-caption` | `#8a7a6a` | 辅助文字、标签 |

#### 语义色（支持地区切换）

设计原则：定义语义 Token，底层颜色根据 `navigator.language` 自动切换。组件只引用语义名，不直接使用 HEX 值。

| Token | 含义 | 中国区 (zh-*) | 欧美区 (其他) |
|---|---|---|---|
| `--color-positive` | 增长/盈利/正向 | `#b95757` (中国红) | `#8b9c7e` (鼠尾草绿) |
| `--color-negative` | 下跌/亏损/负向 | `#8b9c7e` (鼠尾草绿) | `#b95757` (中国红) |
| `--color-danger` | 警示/风险/紧急 | `#c27b6b` (陶土红) | `#c27b6b` (陶土红) |

**实现方式：**

```css
/* design-tokens/src/colors.css */
:root {
  --color-positive: #b95757;  /* 默认中国区 */
  --color-negative: #8b9c7e;
  --color-danger: #c27b6b;
}

:root[lang="en"], :root[data-locale="en"] {
  --color-positive: #8b9c7e;
  --color-negative: #b95757;
}
```

#### 完整色板速查

```

鼠尾草绿 #8b9c7e — Negative (CN)    陶土红 #c27b6b — Danger
暖杏白 #f9f6f0 — Surface    羊毛色 #f0ebe4 — Muted
墨灰   #2c2c2c — Text       辅助灰 #8a7a6a — Caption

```

### 2.3 字体阶梯

优先使用系统原生字体，避免加载额外字体文件（H5 场景加载速度优先）。

| 层级 | 字号 | 字重 | 行高 | 用途 |
|---|---|---|---|---|
| Display | 32px | Bold (700) | 1.3 | 结果页主标题（"你的财务DNA代码"） |
| H1 | 24px | Bold (700) | 1.4 | 页面标题、卡片标题 |
| H2 | 18px | Semibold (600) | 1.5 | 报告内区块标题 |
| Body | 16px | Regular (400) | 1.7 | 正文、题目文案 |
| Label | 14px | Semibold (600) | 1.4 | 按钮文字、Tab 标签 |
| Caption | 13px | Regular (400) | 1.5 | 进度提示、时间戳 |
| Micro | 11px | Medium (500) | 1.4 | 标签、徽章、分类标记 |

**字体栈：**

```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC",
             "Hiragino Sans GB", "Microsoft YaHei", "Helvetica Neue", sans-serif;

font-family: "SF Mono", "Fira Code", "Fira Mono", "Roboto Mono", monospace;  /* 代码/数字 */
```

### 2.4 间距尺（4px 基准）

| Token | 值 | 用途 |
|---|---|---|
| `--space-xs` | 4px | 图标与文字间距 |
| `--space-sm` | 8px | 选项间距、标签间距 |
| `--space-md` | 12px | 卡片内边距（紧凑）、列表项间距 |
| `--space-lg` | 16px | 卡片内边距（标准） |
| `--space-xl` | 24px | 区块间距、页面左右边距 |
| `--space-2xl` | 32px | 大区块分隔 |
| `--space-3xl` | 48px | 页面顶部/底部留白 |

### 2.5 圆角档位

| Token | 值 | 用途 |
|---|---|---|
| `--radius-sm` | 4px | 标签、徽章、小按钮 |
| `--radius-md` | 8px | 输入框、列表项、选项卡片 |
| `--radius-lg` | 12px | 卡片、弹窗、面板 |
| `--radius-xl` | 16px | 大卡片、模态框、主容器 |

### 2.6 暗色模式（预留）

design-tokens 中每种颜色预留 dark 变体，通过 `prefers-color-scheme` 自动切换。Oracle 的夜间使用场景（睡前刷朋友圈）占比高，暗色模式是必要功能。

```css
@media (prefers-color-scheme: dark) {
  :root {
    --color-surface: #1a1a18;
    --color-card: #242320;
    --color-text: #e8e0d5;
    /* ... 其余映射待美术定稿 */
  }
}
```

### 2.7 动效预设（预留）

| Token | 用途 | 参考值 |
|---|---|---|
| `--ease-spring` | 选项切换、按钮反馈 | `cubic-bezier(0.34, 1.56, 0.64, 1)` |
| `--ease-fade-up` | 页面进入、卡片出现 | `cubic-bezier(0.16, 1, 0.3, 1)` |
| `--ease-stagger` | 列表逐项出现 | 每项延迟 50ms |
| `--duration-fast` | 微交互 | 150ms |
| `--duration-normal` | 标准过渡 | 300ms |
| `--duration-slow` | 仪式感动画（"基因测序"） | 2000ms |

## 四、engine-core 设计

### 4.1 设计原则

- **纯函数，零副作用** — 不读文件、不请求网络、不操作 DOM。浏览器和 Node.js 均可用
- **类型即文档** — 所有输入输出完整 TypeScript 类型定义
- **版本单一真相源** — 修改 engine-core，所有消费方同时生效
- **可测试** — 纯逻辑不依赖 UI，单元测试覆盖率目标 95%+

### 4.2 当前模块（Phase 1 · Oracle 开发期）

#### scoring.ts — 人格计分

```typescript
function calculatePersonality(answers: Answer[], questions: QuestionJSON[]): PersonalityResult

interface PersonalityResult {
  code: string;              // "S-I-F-E"（可能含波浪号和斜杠）
  faction: Faction;          // "FE 拓荒者流派"
  typeName: string;          // "远航-拓荒者"
  dimensions: {
    P_SC: number;            // 0-100
    P_IG: number;
    P_FV: number;
    P_EO: number;
  };
}

type Faction = "FE 拓荒者流派" | "FO 离岸者流派" | "VE 征服者流派" | "VO 守夜人流派";
```

#### types.ts — 基础类型

```typescript
type Answer = 'A' | 'B' | 'C' | 'D';

type Dimension = 'SC' | 'IG' | 'FV' | 'EO';

interface QuestionJSON {
  id: string;
  type: Dimension;
  text: string;
  options: {
    label: Answer;
    text: string;
  }[];
}
```

**计分规则：** A=+2（强正向）, B=+1（弱正向）, C=+1（弱反向）, D=+2（强反向）。每题只累加到对应维度。每维度 10 题，P = 50 + (正向-反向)/20 × 50。

### 4.3 扩展路线图

| 阶段 | 模块 | 内容 | 消费者 |
|---|---|---|---|
| **Phase 1** | scoring, types | 人格计分（40题 × 4维度）、基础类型 | Oracle |
| **Phase 2** | simulation, events | 推演引擎（projectAssets, compoundInterest, randomEvent）、随机事件池 | Sandbox |
| **Phase 3** | analytics, i18n | 资产分析（portfolioRisk, fireNumber, withdrawalRate）、多语言框架 | Compass, 全部 |

> **注：** 模拟器参数推算（原 params.ts / B 系列题）已从 Oracle 剥离。财务参数的收集将在 Sandbox 启动前的"财务画像设置"页以更合适的 UI 形式独立完成，具体设计留待 Sandbox 阶段。

## 五、设计钩子（预留但不在 Phase 1 实现）

### 5.1 暗色模式

design-tokens 中所有颜色预留 dark 变体。通过 `prefers-color-scheme` 和手动切换两种方式触发。

### 5.2 动效预设库

design-tokens 提供标准动效曲线和时长。组件通过 CSS 变量引用：

```css
.btn-press { transition: transform var(--duration-fast) var(--ease-spring); }
.card-enter { animation: fade-up var(--duration-normal) var(--ease-fade-up); }
```

### 5.3 内容版本管理

题库、文案、算法均可独立迭代。版本号编码在数据传递协议中，确保向后兼容。

### 5.4 统一埋点 Schema

shared-types 中定义跨产品事件类型：

```typescript
// 所有产品共用的事件类型
type EventName =
  | 'quiz_start'
  | 'quiz_answer'
  | 'quiz_complete'
  | 'result_view'
  | 'result_share'
  | 'sandbox_start'
  | 'sandbox_decision'
  | 'sandbox_milestone'
  | 'sandbox_end'
  | 'compass_asset_add'
  | 'compass_review';
```
