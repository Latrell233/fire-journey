### 3.3 题目切换动画编排

#### 3.3.1 选项入场动画 (当前题)

| 顺序 | 元素 | 动画 | 时长 | 延迟 |
|---|---|---|---|---|
| 1 | 选项 A | fade-up (y: 12px → 0) | 250ms | 0ms |
| 2 | 选项 B | fade-up (y: 12px → 0) | 250ms | 60ms |
| 3 | 选项 C | fade-up (y: 12px → 0) | 250ms | 120ms |
| 4 | 选项 D | fade-up (y: 12px → 0) | 250ms | 180ms |

- 选项出现顺序始终为 A → B → C → D，由上至下。
- 每个选项使用 `ease-fade-up` 曲线。

#### 3.3.2 题目切换动画

```
当前题 → 用户点击选项 → 选项选中动画 (150ms)
                      → 停顿 300ms (让用户看到自己的选择)
                      → 题目文字 y: 0 → -16px, opacity: 1 → 0 (200ms, ease-out)
                      → 选项整体 y: 0 → -16px, opacity: 1 → 0 (200ms, ease-out)
                      → 新题目文字 y: 16px → 0, opacity: 0 → 1 (250ms, ease-fade-up)
                      → 新选项逐项入场 (选项 A/B/C/D 各延迟 60ms)
```

使用 Framer Motion `<AnimatePresence mode="wait">` 实现。

#### 3.3.3 阶段切换动画 (如启用阶段标签)

当阶段标签文案变化时，阶段标签：
- 旧标签：opacity 1 → 0, y: 0 → -8px (150ms)
- 新标签：opacity 0 → 1, y: 8px → 0 (150ms, 延迟 150ms)

### 3.4 特殊状态

#### 3.4.1 最后一题 (第 40 题)

- 用户点击选项后，不显示"下一题"按钮。
- 选中动画完成后，停顿 500ms，然后自动进入基因测序动画 (DnaReveal)。
- 选项区整个区域做一次轻微的 scale (0.98) + opacity (0.6) 过渡，表示"答题完成"。

#### 3.4.2 加载状态 (questions 为空)

```
居中显示 "加载中..."，字号 13px，颜色 #a09080
无其他装饰元素
```

### 3.5 移动端适配

- 进度条固定在页面顶部，z-index: 10。
- 题目和选项区域使用 `flex-1` 自适应剩余高度。
- 选项按钮高度不固定：short 选项约 56px，long 选项自动撑高。
- 触摸目标最小高度 44px (满足无障碍标准)。
- 底部安全区留白至少 20px (iPhone 底部 Home Indicator)。
- 左右边距 24px 固定，不随屏幕宽度变化。
- `-webkit-tap-highlight-color: transparent` 禁用默认点击高亮。

每个维度卡片的比例条在卡片入场后 200ms 开始填充：
- 从 width: 0% → width: P_值%，800ms，ease-fade-up。
- 比例条填充结束后，数字标签做一个轻微的 scale 弹跳 (1 → 1.08 → 1, 300ms spring)。

#### 5.3.3 DNA 代码方块入场

- S 方块: 0ms
- I 方块: 100ms

- E 方块: 300ms
- 每个方块: scale(0.8) → scale(1.05) → scale(1), 400ms spring
